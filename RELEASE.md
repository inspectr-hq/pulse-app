# Release Workflow

Pulse uses GitHub Actions to build and publish macOS releases automatically.

## How to release

1. Create and push a Git tag in the form `v0.3.0`.
2. GitHub Actions picks up the tag and runs the release workflow.
3. The workflow builds the app, packages `Pulse.app`, and creates a GitHub Release zip automatically.

## Tag format

- Use `v` followed by a semantic version, for example: `v0.3.0`
- Do not tag releases with build numbers or extra suffixes
- The app version shown in macOS comes from the tag without the `v` prefix

## Notes

- Release builds are generated by `.github/workflows/release.yaml`
- The build number stays separate from the public version
- The release is created automatically after the tag is pushed
